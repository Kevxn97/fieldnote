# reader mode

## Definition
Reader mode is a documented feature area within Obsidian Web Clipper, presented as a distinct topic alongside clipping, highlighting, and interpreter functions in the browser extension’s help materials.

## Description
In this context, reader mode appears as one of the core capabilities associated with Obsidian Web Clipper. It is listed among the extension’s feature topics rather than as a standalone product, which places it within the broader workflow of working with web pages inside the clipper.

Obsidian Web Clipper is described as a free browser extension for highlighting pages and saving web content directly to an Obsidian vault. Reader mode therefore sits within a tool focused on turning web content into locally saved notes and clippings rather than sending that material to a hosted service.

The documented feature set around reader mode includes nearby concepts such as clipping, highlighting, interpreter features, templates, variables, filters, and template logic. The term is somewhat ambiguous in general software usage, since “reader mode” can refer broadly to simplified page reading views. Here, the confirmed meaning is narrower: it is a named Web Clipper feature area within Obsidian’s documentation.

## Why It Matters
Reader mode matters because it is treated as a first-class capability in the Web Clipper feature set, not as an incidental option. Its inclusion alongside clipping and highlighting indicates that reading state and content capture are closely related parts of the extension’s design.

It also matters because Web Clipper is positioned as local-first and privacy-preserving. Content saved through the extension goes to the user’s Obsidian vault, with no user-data collection or usage metrics. Any reader-mode workflow inside that environment inherits the same local and auditable framing.
