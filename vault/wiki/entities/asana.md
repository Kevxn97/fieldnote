# Asana

## Who Or What It Is
Asana appears as an external service that Claude Managed Agents can post outputs to, using MCP servers as the integration mechanism. The available evidence is brief and does not further define Asana’s broader role or product scope.

## Description
In the described Claude Managed Agents workflows, Asana is one of the systems that can receive outputs generated by an agent. It is mentioned alongside Slack as a destination for posting results after the agent completes a task such as generating a report.

The integration path described is through MCP servers. This places Asana within a tool-connected workflow where developers define tools and environments, and Claude uses those integrations while carrying out production-style tasks.

Asana matters here as part of the “take action” layer of an agent workflow, not just the analysis layer. The example implies that agent results do not have to remain inside the execution environment; they can be sent outward into operational systems used by people or teams.

The evidence does not describe how Asana is configured, what objects are created or updated there, or what permissions and approval controls apply specifically to Asana actions.

## Key Relationships
- **Claude Managed Agents**: can post outputs to Asana as part of an automated workflow.
- **MCP servers**: identified as the mechanism through which outputs can be posted to Asana.
- **Slack**: mentioned alongside Asana as another destination system for agent outputs.
- **Reports**: one cited workflow pattern involves generating reports and posting the outputs to Slack and Asana.
- **Production-style workflows**: Asana is referenced in the context of agents intended for scaled, operational use rather than one-off prompts.

## Open Questions
- What specific Asana actions are supported through the MCP integration?
- Does the integration create tasks, comments, projects, attachments, or some other Asana object?
- What approval or human-review gates apply before Claude posts to Asana?
- How are permissions and authentication handled for Asana through MCP servers?
- Can Asana be used only as an output destination, or also as an input/task source for agents?
