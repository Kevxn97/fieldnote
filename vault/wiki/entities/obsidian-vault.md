# Obsidian vault

## Who Or What It Is
An Obsidian vault is the local destination where Obsidian Web Clipper saves highlighted pages and other clipped web content.

## Description
In the available material, the vault appears as the user-controlled storage location for content captured from the web. Obsidian Web Clipper is described as a browser extension that can highlight pages and save that content directly to an Obsidian vault.

The vault is central to the tool’s privacy model. Clipped content is said to be saved locally to the user’s Obsidian vault, rather than sent to a cloud service described here. That local saving behavior is paired with claims that user data is not collected and usage metrics are not gathered.

The vault also anchors several related clipping features. The surrounding documentation linked from the Web Clipper introduction suggests that clipped material may be shaped by templates, variables, filters, and logic before being saved into the vault, though the exact vault-side structure is not described here.

## Key Relationships
- **Obsidian Web Clipper:** saves web content directly to an Obsidian vault.
- **Highlighted pages:** can be captured by the extension and stored in the vault.
- **Local-first saving:** the vault is explicitly where content is saved locally.
- **Privacy claims:** local saving to the vault is linked to statements that no user data is collected and no usage metrics are gathered.
- **Templates, variables, filters, and logic:** these linked Web Clipper features may affect what gets saved into the vault, though the details are not provided here.
- **Reader and Interpreter features:** these are documented as related Web Clipper capabilities that may influence clipped content before it reaches the vault.
