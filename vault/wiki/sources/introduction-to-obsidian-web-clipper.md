# Introduction to Obsidian Web Clipper

## Summary
This source is a brief Obsidian Help introduction page for Obsidian Web Clipper. It defines Web Clipper as a free browser extension that lets users highlight pages and save web content directly to an Obsidian vault.

The page mainly serves as a navigation hub. It lists official installation locations for Chrome/Chromium browsers, Firefox, Safari, and Edge, and links to topic pages covering clipping, highlighting, reader mode, interpreter features, troubleshooting, templates, variables, filters, and template logic.

It also makes a clear privacy claim: Web Clipper saves content locally to the user’s Obsidian vault, does not collect user data, does not gather usage metrics, and is open source and auditable. This matters because it positions the tool as local-first and transparent rather than cloud-dependent.

## Key Points
- Obsidian Web Clipper is described as a free browser extension for highlighting pages and saving web content to an Obsidian vault.
- Official install links are provided for Chrome Web Store, Firefox Add-Ons, Safari Extensions, and Edge Add-Ons.
- The Chrome listing is explicitly said to cover Chrome, Brave, Arc, Orion, and other Chromium-based browsers.
- The Firefox listing is explicitly said to support Firefox and Firefox Mobile.
- The Safari listing is explicitly said to support macOS, iOS, and iPadOS.
- The help page links to feature documentation for Clip web pages, Highlighter, Reader, Interpreter, Troubleshooting, Templates, Variables, Filters, and Logic.
- The privacy section says content is saved locally to the user’s Obsidian vault.
- The page states that user data is not collected, usage metrics are not gathered, and the codebase is open source and auditable.

## Contradictions And Updates
- This source frames Obsidian Web Clipper as a free browser extension, not as a paid add-on or desktop-only feature.
- It expands platform scope beyond a single browser family by listing Chromium browsers, Firefox, Firefox Mobile, Safari, and Edge.
- It explicitly states that clipped content is saved locally to the user’s vault and that no user data or usage metrics are collected.
- It adds that the code is open source and auditable via the linked GitHub repository.
